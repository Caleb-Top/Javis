# Javis 多模态识别能力开发计划

## 核心原则: 模型只是算力, Javis自己多模态

Javis 的多模态能力不应该依赖 LLM 自带的视觉功能.
LLM(DeepSeek/GPT/Claude)只是大脑的"推理引擎", 
视觉/听觉/触觉应该由 Javis 自己的模块处理, LLM只拿处理后的结构化数据.

```
相机/麦克风 → Javis感知模块 → 结构化数据(文本) → LLM推理 → 行动
                    ↑                                ↓
             本地模型处理图像                  代码执行/工具调用
             (OCR/检测/分类)                   控制电脑
```

---

## 第一阶段: 计算机视觉 (本地, 不依赖LLM)

### 1. OCR 文字识别 (已完成基础)
**技术:** PaddleOCR / Tesseract
```python
from paddleocr import PaddleOCR
ocr = PaddleOCR(use_angle_cls=True, lang='ch')
result = ocr.ocr('screenshot.jpg')
# 输出: [([[x1,y1],[x2,y2],[x3,y3],[x4,y4]], '识别文字', confidence)]
```
Javis 截图后 → OCR提取所有文字+位置 → LLM看到结构化文字版界面

### 2. 屏幕元素检测
**技术:** YOLOv8 界面元素检测
- 检测按钮、输入框、列表、图标等界面元素
- 输出每个元素的类型+位置+尺寸
- 训练数据: 1000张标注的Windows界面截图

### 3. 图标识别
**技术:** ResNet/Swin 分类器
- 识别常见图标: 微信/QQ/Chrome/设置...
- 输出: "坐标(150,200)处是微信图标"
- 不需要LLM看图, 直接给出文字描述

### 4. 条码/二维码扫描
```python
import pyzbar
from PIL import Image
codes = pyzbar.decode(Image.open('screenshot.jpg'))
```

## 第二阶段: 语音处理 (已完成基础)

### 1. 语音识别 (STT)
**当前:** faster-whisper base
**升级:** 
- faster-whisper large-v3 (更高准确率)
- 加上语言自动检测
- VAD语音活动检测, 自动切分长语音

### 2. 语音合成 (TTS)
**当前:** edge-tts
**保留:** edge-tts 质量够好, 网络延迟是问题
**本地备选:** 
- Coqui TTS (完全本地)
- GPT-SoVITS (高质量中文克隆)

### 3. 声纹识别 (新功能)
- 识别说话人身份
- 支持多用户区分
- 用户A说"打开微信" vs 用户B说"打开微信"

## 第四阶段: 多模态融合架构

```
                 ┌─────────────────────────┐
                 │      Javis 大脑(LLM)     │
                 │  只处理结构化文字输入     │
                 └────────┬────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
    ┌─────▼─────┐  ┌─────▼─────┐  ┌─────▼─────┐
    │ 视觉模块   │  │ 语音模块   │  │ 传感器    │
    │ OCR/检测   │  │ STT/TTS   │  │ 键盘/鼠标  │
    │ 人脸/物体  │  │ 声纹      │  │ 系统状态   │
    └───────────┘  └───────────┘  └───────────┘
          │               │               │
    ┌─────▼─────┐  ┌─────▼─────┐  ┌─────▼─────┐
    │ OpenCV    │  │ whisper   │  │ win32api  │
    │ YOLOv8    │  │ Coqui     │  │ psutil    │
    │ PaddleOCR │  │ edge-tts  │  │ pyautogui │
    └───────────┘  └───────────┘  └───────────┘
```

### 关键设计: 感知-推理分离

LLM 永远只接收**文字描述**, 不接收原始图像/音频:

```
❌ 错误方式: 把图片base64发给LLM → 浪费token, 依赖LLM视觉
✅ 正确方式: 本地模型分析图片 → 输出JSON文字 → LLM推理
```

示例:
```python
# 截图后
screenshot() → YOLOv8检测界面元素 → 输出:
"当前界面: QQ音乐播放器
  - 搜索框: 位置(120,80), 文字'输入歌名'
  - 播放按钮: 位置(400,350)
  - 当前歌曲: '普通朋友' - 陶喆
  - 进度条: 3分22秒/4分15秒"

# 送给LLM的是纯文字, 不是图片
# 节省token, 更精确, 不依赖模型能力
```

## 不依赖LLM的多模态 - 为什么是对的

| 对比 | 依赖LLM视觉 | 本地模型处理 |
|------|-----------|------------|
| 速度 | 2-5秒 (API延迟) | 50-200ms (本地GPU) |
| 成本 | 每次看图都花钱 | 免费 |
| 精度 | 模型看图能力有限 | 专用模型更强 |
| 离线 | 不行 | 完全离线 |
| 隐私 | 截图上传云端 | 全部本地 |

Javis 核心架构: **LLM只是大脑, 眼睛/耳朵是自己的.**
