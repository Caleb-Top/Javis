# Javis 智能体 — 缺陷清单 & 改进方向 v1.5

> 基于当前 D:\Javis\ 全部代码 & 运行状态

---

## A. 架构级缺陷

| # | 缺陷 | 现象 | 根因 | 修复方向 |
|---|------|------|------|----------|
| A1 | **无对话记忆** | 每次关浏览器/刷新，Javis 忘了之前聊过什么 | AgentState.messages 只存在内存，无持久化 | 加 ChromaDB 向量存储，每次对话存入，下次启动恢复 |
| A2 | **服务器不持久** | 关掉终端/双击 bat 关了，Javis 就死了 | Python 进程随终端生命周期 | 做成 Windows 服务或 Electron 托盘常驻 |
| A3 | **ReAct 循环脆弱** | LLM 返回格式不对就整个卡死 | 工具调用解析只靠正则，无容错 | 加 JSON Schema 校验 + 自动修正 + 最多重试3次 |
| A4 | **STT 模型首次冷启动慢** | 第一次用语音等了很久 | Whisper base 模型 ~150MB 首次运行时下载 | 安装后预下载，启动时预热 |

## B. 功能级缺陷

| # | 缺陷 | 现象 | 修复方向 |
|---|------|------|----------|
| B1 | **摄像头无预览** | 拍了照但看不到拍了什么 | 前端加 `<video>` 实时预览，拍照后显示缩略图 |
| B2 | **错误信息太技术化** | 用户看到 "WinError 3" 等原始错误 | 每个工具加错误翻译映射，LLM 自然语言解释 |
| B3 | **TTS 缓存不清理** | data/tts_cache/ 无限增长 | 加 LRU 淘汰，最多保留 50 条 |
| B4 | **无打断检测** | Javis 还在说话，你插话它不停 | 麦克风持续监听，检测到你的声音立即停 TTS |
| B5 | **工具调用无确认** | 执行危险操作没提示 | 加分险等级 - 截图级别自动、删除级别弹窗确认 |

## C. 体验级缺陷

| # | 缺陷 | 现象 | 修复方向 |
|---|------|------|----------|
| C1 | **消息无时间戳** | 不知道哪条消息是什么时候发的 | 每条消息加时间标签 |
| C2 | **对话不能导出** | 聊天记录无法保存 | 加"导出对话"按钮 → TXT/Markdown |
| C3 | **无快捷指令** | 每次都要打字 | 加 /screenshot /system /camera 等斜杠命令 |
| C4 | **工具结果太生硬** | 返回 Raw JSON/英文 | LLM 在收到工具结果后翻译成自然中文 |
| C5 | **无多轮上下文理解** | "再试一次"它不知道"再"指什么 | 依靠 A1 记忆系统解决 |

## D. 性能缺陷

| # | 缺陷 | 修复方向 |
|---|------|----------|
| D1 | qwen2.5:7b 推理速度 ~2-4s/次 | 换更小模型(3b)做简单任务，7b 做复杂任务，自动路由 |
| D2 | 粒子动画 80 个 + 连线，低配机器可能卡 | 降为 40 粒子 + 禁用连线（可配置） |
| D3 | WebSocket 无心跳保活 | 加 30s ping/pong，超时重连 |

## E. 安全缺陷

| # | 缺陷 | 修复方向 |
|---|------|----------|
| E1 | API Key 明文存 config.yaml | 加密存储或用 Windows 凭据管理器 |
| E2 | 无操作日志 | 记录所有工具调用，方便回溯 |
| E3 | 无速率限制 | 防止 LLM 死循环无限消耗 token |
