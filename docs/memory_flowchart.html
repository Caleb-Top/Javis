⚠️ 当前缺陷
D1 📍 无作用域标识
所有 Fact 无 scope_id/scope_type
→ 无法按项目/对话/分支隔离记忆
D2 🗜️ 摘要全是全局的
_cycle_summarize() 无视 scope，
所有事实混在一起压缩
D3 📏 检索不感知层级
build_memory_context() 搜索所有事实，
忽视当前在哪个项目/对话/分支
D4 🔗 项目间无互联
没有跨项目记忆/工具共享机制
D5 ⬆️ 没有记忆提升
分支的重要结论不会自动
「升级」到所属对话/项目
D6 📊 用户画像不跨项目
用户偏好/style 写入全局，
不区分项目上下文
🔧 架构级缺陷（深度审计）
D7 🔀 两套平行检索系统
controller.retrieve() 和 retriever.build_memory_context()
独立实现相同功能，结果不一致，无统一入口
D8 🧩 Learner 使用独立 Brain 实例
knowledge/learner.py 自建 Brain() 对象
学到的内容从不进入主记忆系统
D9 🔐 无线程安全
brain._facts 被主线程、自动刷盘线程、
控制器循环线程同时读写，无锁保护
D10 🧠 无语义搜索
全部检索基于关键词匹配（FTS5 + 子串），
无向量嵌入，「修复bug」和「解决错误」不同结果
D11 ⏳ 无遗忘曲线
priority 静态不变，无时间衰减，
不区分工作记忆和长期记忆
D12 📜 consolidator.py 缺失
源码文件不存在，仅存旧的 .pyc 缓存。
合并逻辑分散在 semantic.py/procedural.py/brain.py
