# Javis 项目全面自检报告

**日期**: 2026-07-23  
**审查范围**: D:\Javis 项目所有手写源代码（~130 个 .py/.js/.html/.css/.yaml/.bat/.ps1 文件）  
**审查方法**: 5 个并行 Agent 逐文件、逐行深度审查

---

| 严重程度 | 数量 | 说明 |
|----------|------|------|
| 🔴 严重 | 65+ | 安全漏洞、运行时崩溃、API 不匹配、资源泄漏 |
| 🟡 中等 | 240+ | 异常处理不当、硬编码路径、线程安全、封装破坏 |

## 🔴 严重问题清单（按模块分类）

### 安全漏洞（命令注入 / 路径遍历）

1. **`main.py:501`** — `subprocess.run(cmd, shell=True, ...)` 在 `/api/workspace/terminal` 中将用户输入直接传入 shell，存在远程命令注入漏洞。

2. **`main.py:243`** — `safe_path.replace("..","").replace("~","")` 路径清理过于简单，攻击者可使用 `..../`（替换 `..` 后残留 `../`）绕过，存在路径遍历漏洞。

3. **`system.py:20`** — `subprocess.run(command, shell=True)` 直接将用户/AI agent 提供的命令通过 shell 执行，极其危险的命令注入。

4. **`code_exec.py:297-306`** — `shell=True` 拼接用户命令，可注入 `& del /F /S C:\*` 等命令。

5. **`code_exec.py:336`** — 运行用户编译的 C/C++ 可执行文件，极高风险的任意代码执行。

6. **`code_exec.py:362`** — `exec(cleaned, local_ns)` 执行用户提供的代码作为 handler，无额外安全检查。

7. **`sandbox.py:240-244`** — PowerShell 命令转义仅用 `code.replace('"', '\\"')`，攻击者可注入 `\"` 混淆突破。

8. **`tool_catch2.py:112`** — `shell=True` 命令注入，且硬编码 `D:/Javis` 路径。

9. **`tool_creator.py:20-24`** — 字符串插值生成 Python 代码文件，若 `handler_code` 来自用户输入，写入的代码被执行时存在远程代码执行风险。

10. **`provider_loader.py:89-92`** — `exec(f.read(), namespace)` 执行任意 provider.py 文件，恶意插件可远程执行代码。

11. **`config.yaml:14-16`** — DeepSeek API key 和 GLM API key 以 base64 编码硬编码在配置文件中。Base64 不是加密，任何人可解码获取密钥。

12. **`setup.py:68`** — Token 文件用 `open()` 读取但无 `with` 语句关闭，token 泄露风险。

### 运行时崩溃 / 功能完全不可用

13. **`subagent.py:53-57`** — `_build_filtered_tools` 中 `t.name` 对字符串调用属性，`ToolRegistry.list_all()` 返回 `list[str]` 而非对象列表，导致 `AttributeError`。

14. **`subagent.py:93-98`** — `LLMClient` 没有 `chat()` 方法；`tc.arguments` 应为 `tc["params"]`。子代理系统**完全不可用**。

15. **`subagent.py:125`** — `max_turns=0` 时 `turn` 变量未绑定，`UnboundLocalError`。

16. **`agent.py:468`** — 当 `PERM_LEVELS` 导入失败时，`PERM_LEVELS.get(...)` 抛出 `NameError`。

17. **`agent.py:124`** — `build_dynamic_prompt` 函数名冲突（本地定义覆盖了 `prompt_builder` 的导入），功能重复且行为不一致。

18. **`tool_hello_world.py:7-9`** — 外层 handler 定义了嵌套 handler 但**无 return 语句**，工具完全不可用。

19. **`tool_math_power.py:7-12`** — 同样的嵌套 handler 无 return bug。

20. **`tool_pdf_to_word.py:7-68`** — handler 定义了 `pdf_to_word` 内部函数但**从未调用**，无 return 语句。工具完全不可用。

21. **`cron_scheduler.py:5`** — `import fcntl`，这是 Unix 特有模块，在 Windows 上不存在，整个模块**直接崩溃**。

22. **`cvu.py:252`** — `if __name__ == "__main__"` 块中引用 `sys.argv` 但 `sys` 未在文件顶部导入，`NameError`。

23. **`auto_updater.py:149-167`** — 使用 Unix `tar` 命令做备份/恢复，在 Windows 目标平台上完全不可用。

24. **`gateway/slack_adapter.py:5`**, **`telegram_adapter.py:5`**, **`wechat_adapter.py:5`** — 从 `gateway_manager` 导入不存在的类 `PlatformAdapter`, `Message`, `Reply`，**ImportError**。

25. **`memory/search_api.py:10-32`** — `search_episodes()` 等多个函数调用传入不存在的 `query` 参数，签名不匹配，`TypeError`。

### 功能逻辑严重缺陷

26. **`core/agent.py:601-674`** — `_auto_learn_from_actions` 方法末尾 `except Exception: pass` 吞掉所有异常，brain 写入失败静默。

27. **`core/agent.py:676-719`** — `_after_learn` 同样 `except Exception: pass`，记忆存储失败静默。

28. **`auto_updater.py:272-285`** — `check_update` 返回 `{"success": False}` 时被 `ToolRegistry.execute()` 错误包装为成功结果。

29. **`manifest.py:62,90`** — `register_search_tools` 函数被定义了两次，版本1（使用 `glob_search`）被版本2（使用 `glob_find`）覆盖。

30. **`desktop.py:287`** — `pyautogui.FAILSAFE = False` **禁用故障保护机制**，自动化脚本无法被用户中断。

31. **`_languages/bash.py, go.py, rust.py, test_lang.py`** — `if __name__ == "__main__"` 代码块被重复了 15-20 次，明显的代码生成错误。

32. **`kernel/kernel.py:210,251`** 和 **`sleep_learning.py:32`** — 使用相对路径 `Path("brain_data/...")` 而非基于 `__file__` 的绝对路径。

33. **`kernel/training/train_loop.py:143-150`** — 梯度累积计数器在所有 leaf 之间共享，导致优化器 `zero_grad()` 清掉其他 leaf 的梯度。

34. **`knowledge/brain.py:345-349`** — 压缩逻辑 `f.content = "[压缩] " + summary[:60]` **覆盖原始内容**，与注释"绝不删除"矛盾。

35. **`check_detail.py:10`** — `bytes(rows)` 语法错误，此脚本无法运行。

36. **`web/js/app.js:286`** — `renderMd()` 中正则替换可能破坏已转义的 HTML，存在 XSS 漏洞。

## 🟡 中等优先级问题（代表性样本）

### 异常处理

- **20+ 处裸 `except:` / `except: pass`** — 遍布 controller.py, episodic.py, indexer.py, procedural.py, semantic.py, brain.py, consolidate, script files，吞掉 KeyboardInterrupt 和 SystemExit
- **`llm_client.py:108`** — 裸 `except: params = {}`
- **`tray.py:22,44`** — 裸 except 在低层级键盘钩子中抑制所有异常
- **`engine.py:96-101`** — 备用路径失败时 `_route_history` 未记录

### 硬编码路径/值

- 30+ 处硬编码 `D:/Javis` 或 `C:/Windows/Fonts/simhei.ttf`
- `prompt_builder.py:143-186` — 直接访问 `brain._facts` 私有属性
- `agent.py:70-121` — 超长 System Prompt 与 prompt_builder 重复
- `engine.py:120` — 硬编码路径 `D:\\Javis\\ollama_models`

### 线程安全

- `controller.py` — `_cache` 字典多线程读写无锁
- `gateway_manager.py` — 全局单例 `_gateway` 无锁
- `agent.py:52-55` — `action_log` 模块级可变全局变量
- `memory/indexer.py:34` — `check_same_thread=False` + SQLite 多线程风险

### 资源泄漏

- `cron_scheduler.py:141-147` — `acquire_lock()` 打开 fd 但从未关闭
- `tool_convert_format.py:38,62,75` — fitz/pdf2docx 资源未在异常路径关闭
- `setup.py:68` — Token 文件未用 `with` 关闭
- `config_api.py:40,58` — 配置文件未用 `with` 语句

### 逻辑缺陷

- `memory/controller.py:261-263` — 遍历 `_facts` 时直接 remove，可能跳过元素
- `memory/controller.py:220` — `tick % 10 == 0` 和 `tick % 30 == 0` 重叠触发
- `memory/indexer.py:37` — `PRAGMA synchronous=OFF` 与"数据 0 丢失"注释矛盾
- `task_tools.py:222-239` — 添加依赖关系未检查循环依赖
- `engine.py:80-87` — 降级后备用路径成功不重置 `_consecutive_failures`
